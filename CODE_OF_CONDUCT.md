# Code of Conduct

awfan should be a welcoming, technical, and safety-conscious project.

## Expected behavior

- Be respectful and patient.
- Critique ideas and code, not people.
- Explain hardware risks clearly and without exaggeration.
- Distinguish tested facts from guesses.
- Protect other people's personal information.
- Help newcomers reproduce issues safely.
- Accept maintainer decisions about project scope and hardware safety.

## Unacceptable behavior

- Harassment, threats, discrimination, or personal attacks
- Publishing another person's private information
- Encouraging unsafe hardware writes without clear warnings
- Deliberately misleading users about compatibility or test results
- Spam, trolling, or repeated off-topic disruption
- Posting vulnerability details publicly before a fix is available

## Enforcement

Project maintainers may edit or remove comments, issues, pull requests, or other contributions that violate this policy. Repeated or serious violations may result in a temporary or permanent ban from project spaces.

Report conduct problems privately through the maintainer's GitHub profile. Include links or screenshots when relevant, but do not repost harmful material publicly.

This policy applies to the repository, issue tracker, pull requests, release discussions, and other official awfan project spaces.
