awfan 1.2.0
===========

Added
-----
- Friendly names for firmware profiles in public output.
- A unified mode command for reading and selecting modes.
- A restore command for returning to the previously active firmware mode.
- Temporary control periods with duration and restart-aware restoration.
- Saved custom presets.
- Broker status, restart, repair, and local log commands.
- Redacted JSON diagnostic reports.
- Update backups, post-install checks, and restoration of the previous package
  when a new package fails validation.
- Frontend, core, and broker component-version reporting.

Safety
------
- Temporary control periods are owned by the background broker and survive
  terminal closure.
- Selecting a firmware mode cancels an outstanding temporary-control timer.
- Every hardware-changing command still requires explicit confirmation.

Compatibility
-------------
Validated on Alienware 16X Aurora AC16251. Other systems should begin with
read-only diagnostics and a redacted report.
