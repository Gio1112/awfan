awfan 1.0.1
===========

Added
-----
- Built-in `awfan update --check` command.
- Built-in `awfan update` command for downloading and installing the latest
  stable GitHub release.
- `awfan update --force` for reinstalling the latest stable release.
- SHA-256 verification before an update is installed.
- Automatic extraction and removal of Windows download blocking from the
  verified release package.

Updater behavior
----------------
- Updates are downloaded from the public Gio1112/awfan GitHub Releases page.
- Git and a local repository checkout are not required.
- The updater waits for the current awfan process to exit before replacing the
  executable.
- Version 1.0.1 must be installed once manually. Future stable releases can be
  installed through `awfan update`.

Unchanged safety behavior
-------------------------
- Every hardware-changing command still requires --yes.
- Firmware boost values remain raw inputs rather than percentages or target
  RPM values.
- Profile zero remains diagnostic-only on the tested AC16251.

Known limitations
-----------------
- AWCC WMI access may require an elevated terminal on some systems.
- Direct firmware fan-percent reads return -2 on the tested system, so awfan
  reports nominal RPM utilization from current RPM / nominal maximum RPM.
- Live RPM may briefly exceed the firmware's reported nominal maximum.
- Write support is validated only on Alienware 16X Aurora AC16251 and remains
  experimental.
