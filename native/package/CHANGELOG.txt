awfan 1.1.0
===========

Added
-----
- Native per-user elevated background broker.
- Automatic broker startup at Windows sign-in through Task Scheduler.
- Secure local named-pipe IPC restricted to the current user, Administrators,
  and SYSTEM.
- `awfan broker-status` for checking broker availability.
- Broker restart settings and start-when-available behavior.
- Broker-aware installer, updater, and uninstaller flows.

Changed
-------
- Normal terminals can now run AWCC status, monitoring, profile, and fan-control
  commands without repeated UAC prompts.
- Hardware commands are forwarded to the elevated broker, which launches the
  internal awfan-core executable and relays output and exit status.
- The release package now contains awfan.exe, awfan-core.exe, and
  awfan-broker.exe.
- The installer requests one administrator approval to register the broker.
- Updates may request one administrator approval while replacing the broker.

Security
--------
- The broker accepts only an explicit allowlist of awfan hardware commands.
- Arguments are passed directly to awfan-core.exe without a command shell.
- Remote named-pipe clients are rejected.
- The pipe access control list is scoped to the installing Windows user,
  Administrators, and SYSTEM.
- Existing --yes confirmation remains mandatory for every hardware write.

Optional broker-free mode
-------------------------
CI, portable use, and advanced users may install with:

  .\install.ps1 -NoBroker

Hardware commands may require an elevated terminal in that mode.

Known limitations
-----------------
- The broker cannot run while Windows is asleep; it continues after resume.
- Direct firmware fan-percent reads return -2 on the tested system, so awfan
  reports nominal RPM utilization from current RPM / nominal maximum RPM.
- Live RPM may briefly exceed the firmware's reported nominal maximum.
- Write support is validated only on Alienware 16X Aurora AC16251 and remains
  experimental.
